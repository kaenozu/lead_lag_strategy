# 日米業種リードラグ戦略 - 初心者ガイド

## 📖 このガイドについて

このガイドは、**株取引初心者**が「日米業種リードラグ戦略」を使って利益を得るための手順を説明します。

---

## ⚠️ 重要：リスク警告

**必ずお読みください**

- この戦略は**利益を保証するものではありません**
- **元本割れの可能性があります**
- 過去のパフォーマンスは将来の結果を保証しません
- 余剰資金で取引を行い、生活費は投入しないでください
- このガイドは投資助言ではありません

---

## 🎯 戦略の概要

### 仕組み（やさしく説明）

1. **米国の業種 ETF**の動きを見る
2. **数日前の米国市場**のパターンを分析
3. **日本の業種 ETF**がどう動くか予測
4. 上がりそうな銘柄を**買い**、下がりそうな銘柄を**売り**

### 特徴

- **ホライズン**: デイトレード（朝買って夕方売る）
- **取引頻度**: 毎日
- **必要資金**: 100 万円〜（推奨）
- **目標リターン**: 年率 5-10%
- **目標ドローダウン**: -15% 以内

## 🚀 始め方

### よく使うコマンド（npm）

| コマンド | 内容 |
|----------|------|
| `npm install` | 依存パッケージのインストール |
| `npm run doctor` | 環境・`data/`・`results/` 書き込みの事前チェック |
| `npm run setup` | 初回: データ取得とバックテスト（`backtest_improved.js`） |
| `npm run signal` | 毎日: シグナル生成（`generate_signal.js`） |
| `npm run server` / `npm start` | Web UI（http://localhost:3000。`start` は `server` と同じ） |
| `npm run analysis` | 過去パフォーマンスの分析 |
| `npm test` | 自動テスト（`doctor:ci` 軽量チェック含む） |
| `npm run paper` | ペーパートレードのデモ（仮想取引サンプル・結果は `results/`） |

どのスクリプトが推奨かは **README.md の「どのファイルを使うか」表** を参照してください。

### ステップ 1: 必要なものを準備

1. **証券口座**
   - 日本株 ETF が取引できること
   - 信用取引（空売り）ができること
   - 推奨：SBI 証券、楽天証券など

2. **資金**
   - 最低：50 万円
   - 推奨：100 万円〜
   - 余剰資金で！

3. **PC 環境**
   - Node.js がインストールできること
   - インターネット接続

### ステップ 2: セットアップ

クローンしたフォルダ名に合わせてディレクトリに移動してください（例では `lead_lag_strategy` とします）。

```bash
cd lead_lag_strategy

npm install

# 初回・データ更新: Yahoo から data/*.csv を取得しバックテスト（数分かかります）
npm run setup
# 同等: node backtest_improved.js
```

**なぜ先に `setup` か:** `generate_signal.js` は **ローカルの `data/` の CSV** だけを読みます。`data/` が空のままだとシグナルが出ません。必ず一度 `npm run setup` を成功させてください。

### ステップ 3: 毎日の運用

#### 朝（8:00-9:00）

```bash
npm run signal
# 同等: node generate_signal.js
```

#### ブラウザでシグナルを見る場合（任意）

別ターミナルで `npm run server` を起動し、**http://localhost:3000** を開きます。API が Yahoo から直接データを取るため、CLI の `signal` と同時に必須ではありません。

**開発者向け:** HTTP でサーバー設定を書き換える `POST /api/config` を使う場合のみ、起動前に環境変数 `ALLOW_CONFIG_MUTATION=1` が必要です。通常の利用では不要です。

出力例：
```
📊 買い銘柄（ロング）
ランク  ティッカー  業種           投資額      シグナル値
  1      1618.T     エネルギー資源   100,000 円  17.70
  2      1632.T     証券・商品       100,000 円  8.74
  ...

📉 売り銘柄（ショート）
ランク  ティッカー  業種           投資額      シグナル値
  1      1621.T     医薬品          100,000 円  -276.80
  ...
```

#### 取引実行（9:00-9:30）

1. **買い注文**: 表示された銘柄を指定金額分買う
2. **売り注文**: 表示された銘柄を空売り

#### 夕方（15:00-15:30）

1. **決済**: すべてのポジションを閉じる
2. **記録**: 損益を記録する

## 📊 シグナルの見方

### 出力ファイル

| ファイル | 内容 |
|----------|------|
| `results/signal.json` | 完全なシグナルデータ（JSON） |
| `results/signal_long.csv` | 買い銘柄一覧（CSV） |
| `results/signal_short.csv` | 売り銘柄一覧（CSV） |

### シグナル値の意味

- **大きい正の値**: 強く買い推奨
- **大きい負の値**: 強く売り推奨
- **0 に近い**: 方向性が不明確

---

## 💰 資金管理

### 推奨ポジションサイズ

| 総資金 | 1 銘柄あたり | 銘柄数 | 総エクスポージャー |
|--------|-------------|--------|-------------------|
| 50 万円 | 5 万円 | 10 銘柄 | 50 万円 |
| 100 万円 | 10 万円 | 10 銘柄 | 100 万円 |
| 300 万円 | 30 万円 | 10 銘柄 | 300 万円 |

### リスク管理ルール

1. **1 銘柄 10% ルール**: 総資金の 10% を超えない
2. **ストップロス**: -5% で損切り
3. **ドローダウン制限**: -10% になったら一時停止

## ⚠️ よくある間違い

### ❌ やってはいけないこと

1. **シグナルを無視した自己判断**
   - アプリの指示に従ってください

2. **ポジションサイズの無視**
   - 欲張って大きすぎるポジションは危険

3. **損切りの遅れ**
   - -5% ルールを厳守

4. **週末のポジション持ち越し**
   - 月曜日のギャップリスク

5. **損失を取り返そうとする**
   - 感情取引は禁止

## 🔧 トラブルシューティング

### エラー：データが取得できない

```
解決法:
1. インターネット接続を確認
2. Yahoo Finance の障害情報を確認
3. 時間を置いて再試行
```

### エラー：メモリ不足

```
解決法:
1. 他のアプリを閉じる
2. PC を再起動
3. データ期間を短くする
```

### シグナルが生成されない

```
解決法:
1. npm run setup（または node backtest_improved.js）を再実行して data/*.csv を揃える
2. data/ フォルダに米国・日本の各ティッカー用 CSV があるか確認
3. エラーメッセージを保存して開発者に報告
```
