# 戦略コードの修正サマリー（2026 年 3 月 27 日）

## 概要

日米業種リードラグ戦略のコアコードにおいて、**ルックアヘッドバイアス**を含む複数の重大な問題を発見し、修正しました。

## 発見された問題

### 🔴 重大：ルックアヘッドバイアス

**場所**: `backtest/real.js`, `backtest/improved.js`

**問題内容**:
バックテストにおいて、t 日の日本市場で取引する際、**t 日の米国終値**を使用していた。しかし、日本市場が営業中の t 日には、米国の終値はまだ観測できない（米国市場は閉まっている）。

**修正前**:
```javascript
const retUsLatest = returnsUs[i].values;  // t 日の米国リターン（観測不能）
```

**修正後**:
```javascript
// ルックアヘッドバイアス回避：t-1 日の米国リターンを使用
const retUsLatest = returnsUs[i - 1].values;
```

**影響**:
- バックテスト結果が過大評価されていた
- 修正前：PCA SUB 年率 6.93% → 修正後：3.22%
- より現実的なパフォーマンス数値になった

---

### 🟠 重要：相関行列のサンプル数不足

**場所**: `lib/math/stats.js`

**問題内容**:
データ不足時にエラーで停止し、バックテストが中断されるリスク。

**修正前**:
```javascript
if (n < 2) {
  throw new Error('Need at least 2 samples to compute correlation');
}
```

**修正後**:
```javascript
if (n < 2) {
  logger.warn(`Insufficient samples for correlation: n=${n}, returning identity matrix`);
  return Array.from({ length: m }, (_, i) =>
    Array.from({ length: m }, (_, j) => i === j ? 1 : 0)
  );
}
```

---

### 🟠 重要：分位点計算の不正確さ

**場所**: `lib/portfolio/build.js`

**問題内容**:
`Math.floor()` を使用していたため、意図より少ない銘柄数で取引されていた。

**修正前**:
```javascript
const q = Math.max(1, Math.floor(n * quantile));  // n=17, q=0.4 → 6 銘柄
```

**修正後**:
```javascript
// 論文の「上位 q%」を正確に再現：四捨五入を使用
const q = Math.max(1, Math.round(n * quantile));  // n=17, q=0.4 → 7 銘柄
```

---

### 🟠 重要：取引コストロジックの不明確さ

**場所**: `lib/portfolio/risk.js`

**問題内容**:
コメントと実装が不一致（`÷2` と `×2` が相殺）。

**修正**:
- ロジックを整理し、初期構築時とリバランス時で明確に区別
- ドキュメントを詳細化

---

### 🟢 軽微：入力検証の強化

**場所**: `lib/pca/signal.js`

**修正**:
空の `returnsUsLatest` でエラーをスローする検証を追加。
