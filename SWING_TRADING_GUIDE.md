## 🎯 実運用ステップ

### ステップ 1: 準備（1 週目）

#### 必要なもの
- [ ] 証券口座（日本 ETF 17 銘柄が取引可能）
- [ ] 運用資金（最低¥1,000 万、推奨¥5,000 万〜）
- [ ] Node.js 環境（v18 以上）
- [ ] 本スクリプト一式

#### 証券会社の選定基準
| 基準 | 必須条件 | 推奨条件 |
|------|---------|---------|
| 手数料 | 0.05% 以下 | 0.03% 以下 |
| 取扱い銘柄 | 1617.T〜1633.T | 全 17 銘柄 |
| 取引システム | 同時発注可能 | API 取引対応 |
| 信用取引 | 必須（ショート用） | 貸株料低い |

**推奨証券会社**: 
- 楽天証券（手数料安い、銘柄豊富）
- SBI 証券（手数料安い、システム安定）
- 松井証券（現物・信用両方対応）

### ステップ 2: ペーパートレーディング（2-4 週目）

#### 目的
- 実執行スリッページの計測
- 戦略の動作確認
- 問題点の洗い出し

#### 実施方法
```bash
# 1. 毎朝 9:00 にシグナル計算
node scripts/mean_reversion_swing.js

# 2. シグナル出力を確認
cat results/mean_reversion_swing_analysis.json

# 3. 仮想ポジションを記録
# （スプレッドシートまたは専用ツール）
```

#### 記録項目
| 項目 | 記録内容 |
|------|---------|
| エントリー日 | YYYY-MM-DD |
| エントリー価格 | 寄り付き価格 |
| ロング銘柄 | 3-4 銘柄 |
| ショート銘柄 | 3-4 銘柄 |
| 決済日 | 7 日後 |
| 決済価格 | 引け価格 |
| 損益 | 円ベース |

#### 評価基準
| 指標 | 目標値 | 評価 |
|------|--------|------|
| ネット AR | > 3% | ✅ 合格 |
| 最大 DD | < -25% | ✅ 合格 |
| スリッページ | < 0.05% | ✅ 合格 |

|--------|--------|------|

#### 注文方法

## 🎯 成功の鍵

### やるべきこと ✅
1. **7 日保有を厳守**（欲張らない）
2. **損切り徹底**（-10% で即決済）
3. **記録の徹底**（全ての取引を記録）
4. **レビューの継続**（月次・四半次）
5. **資金管理**（最大 50% ポジション）

### やるべきでないこと ❌
1. **保有期間の延長**（「もう少し」は禁物）
2. **損切りの先送り**（小さな損失が巨大に）
3. **レバレッジ過剰**（3 倍以下に）
4. **他戦略との比較**（一貫性が重要）
5. **パラメータの頻繁変更**（過学習に注意）
