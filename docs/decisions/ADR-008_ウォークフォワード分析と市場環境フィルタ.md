# ADR-008: ウォークフォワード分析と市場環境フィルタの導入

## Status
Accepted（一部課題あり）

## Date
2026-03-27

## Context
第一次改善（パラメータ最適化・セクターフィルタ・損失ストップ）により、過去 1 ヶ月の損失を -65,144 円→-7,461 円に改善したが、まだマイナスだった。

さらなる改善のため、以下の検証を実施：
1. **ウォークフォワード分析**: パラメータの過学習リスク評価
2. **市場環境フィルタ**: 弱気相場での取引抑制

## Decision

### 1. ウォークフォワード分析の実装

**目的**: パラメータの過学習リスク評価と安定性検証

**実装**: `scripts/walkforward_analysis.js`

**設定**:
- 訓練期間：80 日（4 ヶ月）
- テスト期間：30 日（1.5 ヶ月）
- ステップ幅：30 日
- パラメータグリッド：lambdaReg [0.5, 0.7, 0.9], nFactors [1, 2, 3], quantile [0.4, 0.5, 0.6]

**結果**:
```
期間別パフォーマンス:
Period  訓練 SR  テスト SR  差分    勝率 (%)  リターン (%)
     1      1.56      1.43   -0.13      50.0         0.98
     2      1.48     -0.29   -1.77      53.3        -0.23
     3      1.90     -4.54   -6.43      30.0        -1.82

平均シャープレシオ：-1.13
平均勝率：44.4%
過学習リスク：高（差分平均 2.78）
```

**結論**:
- ⚠️ **過学習リスク「高」**: 訓練期間とテスト期間の性能差が大きい
- ⚠️ **パラメータ不安定**: 期間ごとに最適パラメータがバラバラ
- ⚠️ **テスト期間でマイナス**: 一般化性能に課題

### 2. 市場環境フィルタの実装

**目的**: 弱気相場での取引を抑制し、大損失を回避

**実装**: 
- `lib/marketRegime.js`: 市場環境判定モジュール
- `scripts/backtest_with_market_regime.js`: 市場環境フィルタ付きバックテスト

**設定**:
```javascript
{
  lookback: 200,              // 200 日移動平均
  bullThreshold: 1.05,        // 強気：価格/MA ≧ 1.05
  bearThreshold: 0.95,        // 弱気：価格/MA ≦ 0.95
  positionSizeBull: 1.0,      // 強気：100%
  positionSizeBear: 0.0,      // 弱気：0%（取引停止）
  positionSizeNeutral: 0.5    // 中立：50%
}
```

**結果**:
- データ期間不足により十分な検証未能了
- 実装は完了しているが、閾値調整と長期検証が必要

## Alternatives Considered

### パラメータ固定の継続
- **Pros**: 単純、過学習リスク低
- **Cons**: 市場環境変化に適応できない
- **却下理由**: ウォークフォワードでパラメータ不安定が判明

### 機械学習モデルの導入
- **Pros**: 非線形パターン捉え直し
- **Cons**: 過学習リスク大、説明性低下
- **却下理由**: 基礎的な戦略の安定化が先決

### アンサンブル化
- **Pros**: 複数パラメータで安定化
- **Cons**: 計算コスト増、複雑化
- **却下理由**: 過学習リスクが判明したため、根本解決にならない

## Consequences

### 判明した課題

1. **過学習リスクの高さ**
   - 訓練期間のシャープレシオ：1.56〜1.90
   - テスト期間のシャープレシオ：-4.54〜1.43
   - 差分平均：2.78（⚠️ 危険水準）

2. **パラメータの不安定性**
   - lambdaReg: 0.5, 0.7, 0.9 が均等に出現
   - nFactors: 1〜3 がバラバラ
   - 唯一安定：quantile=0.6（100%）

3. **一般化性能の不足**
   - テスト期間でプラスになったのは 1 期間のみ
   - 他はマイナス（最大 -4.54）

### 推奨アクション

1. **戦略の根本的な見直し**
   - 部分空間正則化 PCA の前提再検討
   - 特徴量の追加・改善

2. **データ期間の拡大**
   - 日本 ETF データが 2018 年以降のみ
   - 最低 5 年分の検証が必要

3. **シンプルな戦略への回帰**
   - パラメータ数を減らす
   - 単一パラメータでの長期検証

## Risks and Mitigations

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 過学習 | 高 | パラメータ固定、シンプル化 |
| 一般化性能不足 | 高 | 長期検証、他銘柄での検証 |
| データ不足 | 中 | データソースの追加検討 |
| 戦略の有効性喪失 | 高 | 代替戦略の並行検討 |

## 結論

ウォークフォワード分析により、**過学習リスクが高い**ことが判明した。第一次改善は一時的な損失抑制には効果的だったが、戦略の根本的な有効性は確認できなかった。

**推奨**: 戦略の根本的な見直しと、データ期間を拡大した長期検証を実施する。
